# Postman-коллекция: тестирование API обновления комментария

[![Postman](https://img.shields.io/badge/Postman-v2.1-orange)](https://www.postman.com/)

**Тестовый стенд:** [staff-testing.testkontur.ru](https://staff-testing.testkontur.ru)  
**Swagger API:** [https://staff-testing.testkontur.ru/swagger/index.html](https://staff-testing.testkontur.ru/swagger/index.html)

## Описание

Коллекция предназначена для функционального и негативного тестирования PUT-эндпоинта обновления комментария:

`PUT /api/v1/comments/{id}`

Коллекция покрывает:
- позитивные сценарии (обновление текста, граничные значения длины)
- негативные сценарии (превышение длины, пустой комментарий, невалидные/несуществующие аттачменты, несуществующий комментарий)
- проверки работы с вложениями (attachments)

В ходе тестирования **обнаружены дефекты** в логике работы API (см. раздел «Найденные дефекты»).

## Содержание коллекции

| № | Название запроса | Что проверяет |
|---|------------------|----------------|
| 1 | Текст в комменте без аттачмента | Базовое обновление текста (без вложений) → 200 OK |
| 2 | Граничное значение текст 5000 знаков | Максимальная длина текста → 200 OK |
| 3 | Граничное значение текст 5001 знак | Превышение максимальной длины → 400 Bad Request |
| 4 | Текст и аттачмент оба null | Запрет пустого комментария → 400 Bad Request |
| 5 | Текст + аттачмент | Добавление вложения в комментарий без аттачментов → **баг** |
| 6 | Пустой текст + аттачмент | Пустой текст с вложением → **баг** |
| 7 | Текст - аттачмент | Обновление только текста (поле attachments отсутствует) → **баг** (неоднозначное поведение) |
| 8 | Несуществующий id аттачмента | Передан UUID, который не существует → **баг** |
| 9 | Невалидный id аттачмента | Передана строка, не являющаяся UUID → 400 Bad Request (корректно) |
| 10 | Несуществующий коммент | Обновление комментария с валидным UUID, но несуществующим → 404 Not Found |

> **Важно:** запросы 5–8 выявили отклонения от ожидаемого поведения. Подробности – в разделе «Найденные дефекты».

## Настройка окружения

1. Скачайте файл [`postman_environment.example.json`](./postman_environment.example.json).
2. Переименуйте его в `StaffTesting.postman_environment.json` (или любое другое имя).
3. Откройте файл и заполните свои данные:
   - `accessToken` — Bearer-токен авторизации.
   - `commentWithoutAttachment` — UUID комментария **без** вложений.
   - `commentWithAttachment` — UUID комментария **с** вложениями.
   - `validAttachmentId` — UUID существующего файла-вложения.
4. Импортируйте файл в Postman (Environment → Import).
5. Убедитесь, что в коллекции переменные подставлены (`{{baseUrl}}`, `{{accessToken}}` и т.д.).

> **Примечание:** значения переменных должны соответствовать данным на тестовом стенде.

## Найденные дефекты

В ходе выполнения запросов были обнаружены следующие несоответствия ожидаемому поведению:

| Запрос | Ожидаемый результат | Фактический результат | Статус |
|--------|----------------------|------------------------|--------|
| №5 (Текст + аттачмент) | Комментарий обновлён: текст изменён, **аттачмент добавлен** (код 200) | Код 200, текст обновлён, **поле `attachments` не изменилось** | ❌ Баг |
| №6 (Пустой текст + аттачмент) | Комментарий обновлён: текст пуст, **аттачмент добавлен** | Код 200, текст пуст, **`attachments` не изменилось**; в UI отображается пустой комментарий без вложения | ❌ Баг |
| №7 (Текст - аттачмент) | При PUT-запросе без поля `attachments` список вложений должен сброситься (полная замена ресурса) | Старые вложения остаются | ❌ Баг (либо неоднозначность спецификации) |
| №8 (Несуществующий id аттачмента) | Ошибка валидации (400 Bad Request или 404 Not Found) | Код 200, вложения не меняются | ❌ Баг |

**Дополнительное замечание по №9:**  
При передаче невалидного UUID (`"ыыыы"`) возвращается сообщение с внутренней технической деталью:  
`"[attachments[0]: Error converting value \"ыыыы\" to type 'System.Guid'...]"`.  
Это отличается от пользовательского формата ошибок в других запросах (например, `"Объект не найден"`). Рекомендуется унифицировать стиль сообщений об ошибках.

## Что можно добавить в будущем (расширение коллекции)

- Невалидный UUID комментария (например, `"not-a-guid"`) → ожидается 400
- Обновление чужого комментария (обычный пользователь) → 403 Forbidden
- Обновление чужого комментария (админ) → 200 или 403 (зависит от политик)
- Обновление комментария после выхода пользователя из сообщества
- Отсутствие авторизации → 401 Unauthorized
- Просроченный / битый токен → 401
- Передача существующего ID, но принадлежащего не файлу (например, другому комментарию)
- Слишком много файлов (превышение лимита)
- Дубликаты одинаковых attachment ID
- Разница между отсутствием поля, `null`, `""`, `"   "`
- Пустое тело запроса
- Спецсимволы в тексте (эмодзи, иврит, арабский, управляющие символы)
- Проверка поля `modifiedAt` – должно обновляться при любом изменении
